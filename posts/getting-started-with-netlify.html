Getting Started with Netlify

GitHub Pages is a simple and popular service by GitHub which publishes a website from your GitHub repository.

But, Netlify provides much more control and features. Netlify is more powerful than GitHub Pages because it lets you losslessly compress images, bundle and minify CSS and JavaScript, manage DNS and much more, all for free. Here is our article showing you how to deploy a website with GitHub Pages, if you would like to see that.

Here, you will learn how to deploy a website with Netlify, a very powerful way to publish a website from a Git repository.

1. Create an account

To start using Netlify, you'll first need to create an account. Click here to create a Netlify account.

You will see this screen when you click the link. If you have a GitHub, GitLab or Bitbucket account, you can use one of them to sign up for Netlify. If you don't have any of these accounts or don't want to use one of them, you can use your email to sign up.
Once you sign up, you may need to verify your email, so check your email inbox, open the verification email and click the verification link.
Once you have verified your email, you will see a form asking for your name and what kind of website you want to build. You can fill them if you want to, or click the skip button at the bottom to skip these steps.
After doing these, you will now see the Netlify dashboard with a modal dialog with a quick-start guide.

2. Create a new site

Now, you are ready to publish your site.

In this section of the Netlify dashboard, you can create a new site.

To create a new site with Git, click the "New site from Git" button. Netlify also has the option to create a site without Git, where you can make a folder of your site and drag-and-drop it on the area provided.

This is what you'll see when you click "New site from Git". Here you can choose which Git provider to use for your site. You need to choose the provider which is hosting your website repository. For example, if your site is on GitHub, click the GitHub button.
Once you choose the provider you need, you'll see a popup asking you to log in with the account for your Git provider. Once you log in and authorize Netlify, you'll see a list of your repositories.

Choose your site's repository from the list, and then you'll see a screen like this:

Select the branch that your site deploys from. If you use any static site generator or build tool, type its build command too. Once everything is done, click the "Deploy site" button at the bottom.

Now, your site will deploy in a few seconds! But first, we need to configure it a little.

3. Configure your site

The site name needs to be changed, because by default, Netlify just uses a few random words and numbers for your site name.
Click the Domain settings button at the top to change the site's name.

Click "Options" and click "Change site name". Now you can change your site's name to whatever you like.

Your site is now ready to go!

Your site will now be accessible at yoursitename.netlify.app where "yoursitename" is the name you chose in step 3. If you want to use a custom domain, you can add one in the same Domain settings area in Netlify by clicking "Add custom domain". Follow Netlify's instructions to use the domain on your site.
